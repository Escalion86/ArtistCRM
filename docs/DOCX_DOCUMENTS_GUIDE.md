# DOCX: Договоры и Акты в ArtistCRM

## Что умеет система

- Формирует `Договор` и `Акт` в `.docx` из шаблонов.
- Подставляет переменные из карточки мероприятия, клиента и настроек реквизитов артиста.
- Поддерживает:
  - пользовательские DOCX-шаблоны;
  - стандартные DOCX-шаблоны (fallback).

## Где включается в интерфейсе

1. `Настройки -> Работа с документами`
2. Загрузите шаблоны:
   - `DOCX-шаблон договора`
   - `DOCX-шаблон акта`
3. При необходимости скачайте стандартные шаблоны:
   - `Скачать стандартный шаблон` (для договора/акта)

## Ограничение по тарифу

- Функционал доступен только если тариф разрешает `Работа с документами` (`allowDocuments = true`).
- Если доступа нет:
  - блок `Работа с документами` скрыт в `Настройках`;
  - в редакторе мероприятия скрыты кнопки `Сформировать договор` и `Сформировать акт`;
  - скрыты блоки ссылок на договоры/акты/счета/чеки.
- Чекбокс `По договору` в мероприятии виден всегда.

## Где хранятся шаблоны

Пользовательские шаблоны сохраняются в `SiteSettings.custom`:

- договор:
  - `contractDocxTemplateBase64`
  - `contractDocxTemplateFileName`
- акт:
  - `actDocxTemplateBase64`
  - `actDocxTemplateFileName`

Стандартные шаблоны (fallback) берутся из:

- `public/templates/default-contract-template.docx`
- `public/templates/default-act-template.docx`

## Специальные переменные (важно)

### `{РЕКВИЗИТЫ_СТОРОН}`

В DOCX заменяется на табличный блок:

- слева таблица `Исполнитель` (реквизиты артиста),
- справа таблица `Заказчик` (реквизиты клиента).

Пустые значения оставляются пустыми (без `_______`).

### `{ПОДПИСИ_СТОРОН}`

В DOCX заменяется на таблицу `2x2` без рамок:

- `1,1` — `Исполнитель`
- `1,2` — `Заказчик`
- `2,1` — `Фамилия И.О. исполнителя + "_______________"`
- `2,2` — `Фамилия И.О. заказчика + "_______________"`

## Имена итоговых файлов

- Договор: `Договор №<номер> от <dd.mm.yyyy>.docx`
- Акт: `Акт №<номер> от <dd.mm.yyyy>.docx`

## Где задаются реквизиты артиста

- `Настройки -> Работа с документами -> Редактировать реквизиты`
- Юр. статус артиста:
  - `Индивидуальный предприниматель`
  - `Самозанятый`

Для режима `Самозанятый` поле `ОГРНИП` не обязательно.
