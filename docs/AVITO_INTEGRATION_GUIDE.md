# Avito Integration Guide

## Назначение

Интеграция Avito принимает новые сообщения из Avito Messenger и создает в
ArtistCRM заявку со статусом `draft`.

Интеграция персональная: каждый пользователь ArtistCRM подключает свой аккаунт
Avito в `Настройки -> Интеграции -> Avito`.

## Что нужно со стороны Avito

1. Профессиональный аккаунт Avito или аккаунт с доступом к API.
2. Доступ к Avito API / Messenger API.
3. `Client ID`.
4. `Client Secret`.
5. Доступ к webhook новых сообщений или возможность вручную указать webhook URL.

Если в кабинете Avito нет раздела API, нужно обратиться в поддержку Avito и
запросить доступ к API для собственной CRM-интеграции, включая Messenger API.

## Где искать настройки в Avito

В интерфейсе Avito названия разделов могут отличаться. Обычно путь выглядит так:

1. Войти в аккаунт Avito.
2. Открыть `Профиль`.
3. Перейти в `Настройки`.
4. Найти раздел `Для профессионалов`, `API`, `Интеграции и API` или
   `Собственная разработка`.
5. Создать приложение/ключ API.
6. Скопировать `Client ID` и `Client Secret`.
7. Проверить, что приложению разрешены чаты, сообщения и webhook.

## Настройка в ArtistCRM

1. Открыть `Настройки -> Интеграции`.
2. Раскрыть блок `Avito`.
3. Вставить `Avito Client ID`.
4. Вставить `Avito Client Secret`.
5. Если Avito показывает `User ID` / `Account ID`, вставить его в поле
   `Avito User ID`.
6. Нажать `Подключить`.

ArtistCRM проверит учетные данные через Avito token endpoint и сформирует
персональный webhook URL:

```text
https://YOUR_DOMAIN/api/integrations/avito/webhook/avito_...
```

Если автоматическая регистрация webhook доступна для аккаунта Avito, ArtistCRM
попробует зарегистрировать этот URL сам.

Если ArtistCRM показывает статус `Нужно проверить webhook`, скопируйте
`Адрес webhook Avito` и вставьте его вручную в настройках Avito API.

## Что происходит при новом сообщении

1. Avito отправляет webhook в ArtistCRM.
2. ArtistCRM находит пользователя по персональному webhook token.
3. Первое сообщение в новом чате создает заявку со статусом `draft`.
4. Источник заявки: `Avito`.
5. Переписка сохраняется в `AvitoConversations`.
6. Все сообщения сохраняются в `AvitoMessages`.
7. Повторные сообщения из того же `avitoChatId` не создают дубликаты заявок.
8. Raw payload Avito сохраняется в `event.clientData.lead.raw` и в сообщении.

Avito часто не передает телефон клиента, поэтому заявка может быть создана без
карточки клиента. Контакт можно дополнить вручную после переписки.

## Переписка и ответы

В карточке Avito-заявки и карточке клиента доступен блок `Переписка Avito`.

В нем можно:

- видеть несколько Avito-чатов, связанных с одной заявкой или клиентом;
- переключаться между чатами по разным объявлениям;
- читать входящие сообщения;
- отправлять ответ в Avito из ArtistCRM.

Если один и тот же клиент пишет по разным объявлениям, ArtistCRM старается
связать чаты с одним клиентом по `avitoUserId` или телефону. Если автоматической
связи недостаточно, переписку можно привязать к клиенту через защищенный API
обновления conversation.

Важно: сообщения, отправленные напрямую из кабинета Avito, попадут в ArtistCRM
только если Avito пришлет webhook по этому событию. Для полной сверки истории
нужно отдельное polling/API-подтягивание истории чатов.

## Проверка

1. Подключить Avito в ArtistCRM.
2. Открыть активное объявление Avito.
3. С другого аккаунта написать тестовое сообщение.
4. Проверить, что в ArtistCRM появилась новая заявка со статусом `draft` и
   бейджем `Avito`.
5. Отправить второе сообщение в тот же чат.
6. Проверить, что новая заявка-дубль не появилась.

## Диагностика

В блоке `Avito` отображаются:

- текущий статус подключения;
- последняя ошибка;
- дата последнего принятого webhook;
- адрес webhook.

Кнопка `Проверить` повторно проверяет `Client ID` и `Client Secret`.

## Безопасность

- Учетные данные Avito хранятся в персональных настройках пользователя.
- Webhook URL содержит персональный случайный token.
- Webhook ищет tenant по token и не принимает события для отключенной интеграции.
- В логах не должны появляться `Client Secret`, токены Avito и полные
  персональные данные клиента.
